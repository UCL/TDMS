55  /**
56  * @brief Performs an optimisation step in a 2D simulation (J_tot==0) when we
57  * have either TE or TM, but not both.
58  *
59  * In the J_tot==0 2D version; the 'TE' case involves components Ey, Hx and
60  * Hz, whilst the 'TM' case involves components Ex, Ez and Hy.
61  *
62  * The idea is to use an alternative upper limit to the loop over j when we
63  * have J_tot==0. As it stands, we are planning the loops listed below. We use
64  * the syntax (k k_min k_max : j j_min j_max : i i_min i_max) to represent the
65  * nested for loop: for( k = k_min; k < k_max; k++) { for( j = j_min; j <
66  * j_max; j++) { for( i = i_min; i < i_max; i++) {
67  * }
68  * }
69  * }
70  *
71  * Exy: Not involved in 2D, (k 0 K_tot+1 : j 1 J_tot : i 0 I_tot)
72  * Exz: TM, (k 1 K_tot : j 0 J_tot+1 : i 0 I_tot)
73  * Eyx: TE, (k 0 K_tot+1 : j 0 max(J_tot, 1) : i 1 I_tot)
74  * Eyz: TE, (k 1 K_tot : j 0 max(J_tot, 1) : i 0 I_tot+1)
75  * Ezx: TM, (k 0 K_tot : j 0 J_tot+1 : i 1 I_tot)
76  * Ezy: Not involved in 2D, (k 0 K_tot : j 1 J_tot : i 0 I_tot+1)
77  * Hxy: Not involved in 2D, (k 0 K_tot : j 0 J_tot : i 0 I_tot+1)
78  * Hxz: TE, (k 0 K_tot : j 0 max(J_tot,1) : i 0 I_tot+1)
79  * Hyx: TM, (k 0 K_tot : j 0 J_tot+1 : i 0 I_tot)
80  * Hyz: TM, (k 0 K_tot : j 0 J_tot+1 : i 0 I_tot)
81  * Hzx: TE, (k 0 K_tot+1 : j 0 max(J_tot,1) : i 0 I_tot)
82  * Hzy: Not involved in 2D, (k 0 K_tot+1) : j 0 J_tot : i 0 I_tot)
83  *
84  * We see that in all cases, the TE update has the following loop on j:
85  * for(j=0;j<max(J_tot,1);j++)
86  * whilst the TM case has:
87  * for(j=0;j<(J_tot+1);j++)
88  *
89  * So we can create variables
90  * int J_loop_upper_bound_plus_1, J_loop_upper_bound
91  * which would take the following values
92  * 3D:
93  * J_loop_upper_bound_plus_1 = J_tot + 1;
94  * J_loop_upper_bound = J_tot;
95  * 2D:
96  * TE:
97  * J_loop_upper_bound = 1;
98  * Not TE:
99  * J_loop_upper_bound = 0;
100  * TM:
101  * J_loop_upper_bound_plus_1 = 1;
102  * Not TM:
103  * J_loop_upper_bound_plus_1 = 0;
